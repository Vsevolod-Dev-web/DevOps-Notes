Для того, чтобы монтировать ФС на постоянной основе, необходимо добавить запись в конфигурацию системы. [/etc/](Стандарт%20FHS)

Конкретный конфиг для постоянного монтирования - `/etc/fstab`

Содержание моего файла-конфига.
```bash
# /etc/fstab: static file system information.
#
# Use 'blkid' to print the universally unique identifier for a
# device; this may be used with UUID= as a more robust way to name devices
# that works even if disks are added and removed. See fstab(5).
#
# <file system> <mount point>   <type>  <options>       <dump>  <pass>
# / was on /dev/nvme0n1p2 during curtin installation
/dev/disk/by-uuid/6fe024d8-c735-4420-9fd1-5545dd3b70a3 / ext4 defaults 0 1
# /boot/efi was on /dev/nvme0n1p1 during curtin installation
/dev/disk/by-uuid/D657-8FFB /boot/efi vfat defaults 0 1
/swap.img       none    swap    sw      0       0
```

Каждая запись состоит из 6 полей:
	Пример:
```bash
UUID=abc123   /mnt/data   ext4   defaults   0   2
```
При постоянном монтировании в первом поле лучше указывать **[UUID](Команды%20для%20монтирования)** (Universally Unique Identifier) - уникальный идентификатор который присваевается разделу при форматировании и никогда не меняется, вне зависимости от того, в какой слот вставлен диск.

1. Название устройства
2. [Точка монтирования](Точка%20монтирования%20(Mount%20point))
3. Файловая система
4. Опции монтирования (defaults = стандартный набор: rw, auto и т.д.)
5. Флаг утилиты резервного копирования DUMP. 0 = не бэкапить, 1 = бэкапить. (Сейчас почти всегда ставят 0, dump устарел.)
6. **fsck** (file system check) - проверка диска на ошибки после некорректного выключения. (0 = не проверять (swap, виртуальные ФС - их проверять бессмыслено) 1 = проверить первым (всегда корень `/`) 2 = проверить после корня (остальные диски))
