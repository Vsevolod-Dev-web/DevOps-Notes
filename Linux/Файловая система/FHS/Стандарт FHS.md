FHS - это стандарт файловой системы в Linux. Он определяет базовую структуру каталогов и их назначений.

| Директория | Описание                                                                | Примеры содержимого                                                  | Релевантность для DevOps                                                     |
| ---------- | ----------------------------------------------------------------------- | -------------------------------------------------------------------- | ---------------------------------------------------------------------------- |
| `/`        | [Корень](Корень) всей файловой системы.                                 | Всё дерево.                                                          | Базовая точка монтирования; здесь монтируют другие FS.                       |
| `/bin`     | [Бинарные файлы](Бинарный%20файл) основных команд (essential binaries). | `ls`, `cp`, `mv`, `bash`.                                            | Критично для boot и recovery; команды всегда доступны.                       |
| `/boot`    | Файлы для загрузки системы.                                             | `vmlinuz` (ядро), `initramfs`, GRUB.                                 | В DevOps: настройка bootloaders, обновление ядер.                            |
| `/dev`     | Устройства (device files).                                              | `/dev/null`, `/dev/sda`, `/dev/zero`.                                | Автоматизация: скрипты для дисков, контейнеры (Docker) монтируют /dev.       |
| `/etc`     | Конфигурационные файлы системы.                                         | `/etc/passwd` (пользователи), `/etc/fstab`, `/etc/nginx/nginx.conf`. | Золотой стандарт DevOps: CI/CD для конфигов, Ansible/Puppet управляют здесь. |
| `/home`    | Домашние директории пользователей.                                      | `/home/user/.ssh`, личные файлы.                                     | Пользовательские данные; в DevOps: настройка окружений, SSH-ключи.           |
| `/lib`     | Библиотеки для /bin и /sbin.                                            | Shared libraries (.so).                                              | Зависимости для системных утилит; важно для контейнеризации.                 |
| `/media`   | Точки монтирования для съёмных носителей (автоматически).               | `/media/usb`.                                                        | Редко в DevOps, но для бэкапов.                                              |
| `/mnt`     | Временные точки монтирования (ручные).                                  | Пустая по умолчанию.                                                 | В скриптах: монтирование NFS/S3 для тестов.                                  |
| `/opt`     | Опциональное ПО (third-party).                                          | `/opt/google/chrome`.                                                | Установка софта без пакетов; в DevOps: custom apps.                          |
| `/proc`    | Виртуальная FS с инфой о процессах и ядре.                              | `/proc/cpuinfo`, `/proc/<pid>/status`.                               | Мониторинг: top/ps читают здесь; Prometheus/Grafana для metrics.             |
| `/root`    | Домашняя директория root.                                               | Конфиги root.                                                        | Root-доступ в скриптах; избегай в production.                                |
| `/run`     | Временные данные runtime (в памяти).                                    | PID-файлы, сокеты ( `/run/docker.sock` ).                            | DevOps: systemd, Docker сокеты; эфемерные данные.                            |
| `/sbin`    | Системные бинарники для админов.                                        | `fdisk`, `mkfs`, `reboot`.                                           | Управление дисками/сетью; скрипты автоматизации.                             |
| `/srv`     | Данные для сервисов (www, ftp).                                         | `/srv/www`.                                                          | Веб-серверы: Nginx/Apache хранят сайты здесь.                                |
| `/sys`     | Виртуальная FS для hardware и kernel params.                            | `/sys/class/net`.                                                    | Tuning: sysctl для оптимизации; мониторинг устройств.                        |
| `/tmp`     | Временные файлы (очищается при reboot).                                 | Temp данные.                                                         | Скрипты: временные файлы; избегай для persistent data.                       |
| `/usr`     | Пользовательские программы и данные (read-only).                        | `/usr/bin` (доп. команды), `/usr/share`.                             | Большинство пакетов здесь; в DevOps: /usr/local для custom.                  |
| `/var`     | Переменные данные (логи, спулы).                                        | `/var/log/syslog`, `/var/lib/docker`.                                | Ключ для DevOps: логи (ELK stack), базы данных, кэши.                        |
