В [функцию](Функции) при объявлении не передаются ожидаемые аргументы\параметры. Они указываются в теле функции. Как бы, в теле функции, мы показываем в какой параметр и где должен попасть какой аргумент.

Bash автоматически раскладывает всё по **нумерованным переменным**:
- `$0` — сам скрипт (`./deploy.sh`)
- `$1` — первый аргумент (`production`)
- `$2` — второй аргумент (`8080`)
- `$@` — все аргументы разом (`production 8080`)
- `$#` — количество аргументов (`2`)

Это автоматическое объявленные bash'ом пронумерованные [переменные](Переменные). Их можно переобъявить с более понятным наименованием. Например: `ENV=$1 PORT=$2`

Возьмём пример, у нас есть какой-то скрипт deploy.sh. Вызываем его и, как и в любую команды, передаём аргументы\параметры:
```bash
./deploy.sh staging 3000 verbose
```

Буквально, тоже самое:
```bash
ls -l /
```

И для небольшой проверки:
```bash
#!/bin/bash
echo "Скрипт: $0"
echo "Окружение: $1"
echo "Порт: $2"
echo "Всего аргументов: $#"
```

Получим:
```bash
Скрипт: deploy.sh
Окружение: staging
Порт: 3000
Всего аргументов: 3
```
