Дескриптор - неотрицательное целое число, присваеваемое каждому ресурсу [процесса](Процесс) (файл, сокет, канал, устройство), чтобы ядро понимало, с чем нужно произвести действие (оправка чего-то в интернет, запись на диск и т.п.)

Грубо говоря, файловый дескриптор помогает направлять данные. Фактически это просто номер, передавая данные на который система делает определенные действия с этими данными. При этом, например `ls` когда мы направляем данные файл file.txt с помощью `>` не знает про дескриптор. Он просто отправляет данные на номерок 1.

У каждого нового процесса в Linux уже открыты три стандартных дескриптора:

- **0 - stdin** (Стандартный ввод). Обычно это клавиатура или то, откуда процесс получает данные.
- **1 - stdout** (Стандартный вывод). Сюда процесс пишет обычные результаты (например, в терминал).
- **2 - stderr** (Стандартный вывод ошибок). Сюда процесс пишет сообщения об ошибках.

Когда ты вводишь `ls -l`, программа `ls` берёт дескриптор 1 и выводит список файлов в терминал. Если случается ошибка, она отправит её в дескриптор 2.

Каждый процесс хранит свои открытые дескрипторы в специальной директории внутри [`/proc`](Стандарт%20FHS.md):

```bash
# Посмотрим дескрипторы текущего shell (например, PID=$$ — это PID текущего bash)
ls -l /proc/$$/fd/
```

```bash
lrwx------ 1 user user 64 Feb 14 12:00 0 -> /dev/pts/0
lrwx------ 1 user user 64 Feb 14 12:00 1 -> /dev/pts/0
lrwx------ 1 user user 64 Feb 14 12:00 2 -> /dev/pts/0
lr-x------ 1 user user 64 Feb 14 12:00 3 -> /home/user/.bashrc
```


```!
Количество открытых дескрипторов на один процесс ограничено, чтообы один "плохой" процесс не съел все ресурсы ядра
```
