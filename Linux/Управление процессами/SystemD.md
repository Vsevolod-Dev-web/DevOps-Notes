#### SystemD
**SystemD** - демон, которому ядро передаёт управления после полной инициализации на kernel level. Systemd имеет [PID](Идентифекаторы%20проесса) 1.

Главная задача systemd - инициализировать всё необходимое для работы системы в user space обращаясь к [unit-файлам](unit-файл).

*systemd*, после того как ядро его запустило, начинает идти по [.target](unit-файл) файлам в установленном порядке, запуская все указанные службы. Где .target файлы - это просто набор symlinks на unit-файлы служб в директории. Как правило *.watns/ (multi-user.target.wants).* Любую службу можно добавить в один из наборов .target чтобы она запускалась на определённом этапе инициализации системы.

Порядок запуска .target файлов определяется цепочкой формируемой из необходимых служб перед запуском текущей. Она определяется директивой `After=` в категории [Unit] юнит-файла.

Для работы с SystemD используется инструмент [`systemctl`](systemctl.md)
