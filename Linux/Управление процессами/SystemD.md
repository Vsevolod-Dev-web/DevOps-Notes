#### SystemD
**SystemD** - демон, которому ядро передаёт управления после полной инициализации на kernel level. Systemd имеет [PID](Идентифекаторы%20проесса) 1.

Главная задача systemd - инициализировать всё необходимое для работы системы в user space обращаясь к [unit-файлам](unit-файл).

Для определения будет ли запускаться какой-либо сервис автоматически SystemD формирует символьную ссылку на unit-файл сервиса. 

Для работы с SystemD используется инструмент [`systemctl`](systemctl.md)
