Я SSH Hardening (упрочнение\закалка SSH) - это процесс конфигурации службы для максимального повышения безопасности удаленного доступа.

SSH полностью настраивается в `/etc/ssh/sshd_config`

!!! Важно !!!

В директории `/etc/ssh/sshd_config.d/` лежат файлы конфига, которые переопределяют основной конфиг `/etc/ssh/sshd_config`.

Если в `sshd_config` какая-то директива включена, а в `/etc/ssh/sshd_config.d/` конфиге она включена. Она будет включена.
## Главные правила SSH Hardening
### Отключить аутентификацию по ssh через пароль.

Самый главный вектор атаки на устройства. Пароль легко подобрать, ключ - нет. 
Поэтому, для закрытия этой дыры, мы выключаем авторизацию по ssh с паролем:

`/etc/ssh/sshd_config`
```bash
PasswordAuthentication yes # Меняем yes на no.
```

### Отключить авторизацию под root через SSH
Авторизация под root через ssh - это тоже самый популярный вектор атаки. Если злоумышленник получит доступ под [root](Пользователи) - вся система его.


`/etc/ssh/sshd_config`
```bash
PermitRootLogin yes # Выключаем эту директиву, заменив yes на no.
```

Чтобы проверить, отключены эти директивы, используй: 

```bash
ssh -o PasswordAuthentication=yes -o PubkeyAuthentication=no vsevolod@ip
```

Флаг `-o` позволяет подключатся, применяя определенные директивы со стороны клиента. При этом, последнее слово всегда за сервером. Если авторизация по паролю отключена на сервере, с этой директивой от клиента зайти не даст.
