**SSH** (Secure Shell) - сетевой протокол прикладного уровня, предоставляющий возможность удалённого управления операционной системой и передачи файлов между компьютерами в незащищенных сетях.

#### Подключение через SSH под капотом
При подключении к устройству через SSH

При попытки подключится по SSH к удалённому ПК, происходит следующее:
1. Устанавливается соединение. SSH сначала договаривается о шифровании канала по незащищенному каналу. 
2. Аутентификация. В зашифрованном соединении проводится аутентификация. Клиент проверяет (через публичный ключ или пароль) с помощью публичного ключа, с тем ли установлено соединение.

"[Передать ключ на сервер](Команды%20SSH)" - положить *.pub в директорию `./.ssh/authorized_keys`
#### Асимметричное шифрование
Принцип работы.
1. **Генерируется 2 ключа:**
	1. **Публичный ключ**
	2. **Приватный ключ**

Публичный ключ - шифрует. Приватный ключ - расшифровывает. 

При подключении по SSH, генерируется 2 ключа. Первый ключ, публичный, передаётся на устройство, *на нужного пользователя.*

Далее оба подключенных через SSH устройства будут использовать этот ключ для зашифровки сообщений.

Когда мы получаем данные, зашифрованные публичным ключом, мы можем расшифровать их приватным.

Так же, [сервер](SSH-Server) проверяет: все верно? - если да, доступ получен. Сервер никогда не видит твой приватный ключ. Он может только спросить, можешь ли ты приватным расшифровать, то что он зашифровал твоим публичным.
