### Два типа пользователей
В Linux есть два типа пользователей:
- **Реальные пользователи** - люди. Ты, я, коллеги на сервере. Они входят в систему через терминал или SSH, у них есть BASH, домашняя директория.
- **Системные пользователи** - не люди, а сервисы. `www-data` (nginx), `postgers` (PostgreSQL), `systemd-network` и т.д. Они созданы только для того чтобы запускать процессы от их имени - ничего больше.

Системные пользователи - абстракция, позволяющая определять доступ к файлам для этого сервиса.

Стоит заметить, что ядро Linux не различает что за пользователь сидит. Это человек, бот или сервис - ядро смотрит на UID и сравнивает с правами файла.

Фактически, различие между сис. пользователями и реальными - в том, что у системных пользователей нету пароля и заблокирован вход.

Данные о текущих пользователях находится в [файле](Стандарт%20FHS): `/etc/passwd`

```bash
www-data:x:33:33:www-data:/var/www:/usr/sbin/nologin

root:x:0:0:root:/root:/bin/bash

vsevolod:x:1000:1000::/home/vsevolod:/bin/bash
```

Каждая строка в этом файле - запись о пользователях, разделённая `:` на 7 секций.

1. Имя пользователя.
2. `x` - Раньше в этой секции хранился пароль. Теперь пароли вынесены в `/etc/shadow`
3. `1000` . Это **UID** - уникальный числовой идентификатор пользователя.
4. `1000`. Это **GID** - идентификатор основной [группы](Группы.md) пользователя.
5. Пустое. Это комментарий\описание пользователя.
6. Личная директория пользователя.
7. **SHELL** - оболочка доступная пользователю для работы.

У `root` UID и GID всегда  `0`. Ядро Linux определяет суперпользователя именно по UID = 0.
