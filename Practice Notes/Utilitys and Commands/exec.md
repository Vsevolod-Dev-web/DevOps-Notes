Самое главное, стоит понимать, что exec - это изменение текущей оболочки.

`exec` - это встроенная функция оболочки (shell), которая заменяет текущий процесс оболочки (нап. терминал) новым процессом.

В отличии от обычного запуска процесса из терминала, когда создаётся [дочерний процесс](Процесс) под терминалом, запуск через exec завершает текущий процесс оболочки и ставит запущенный процесс на его место. 

Под "ставит запущенный процесс на его место" стоит понимать, что новый процесс перенимает следующие параметры:
- [Файловые дискрипторы](Дескриптор)
-  [PID](Идентифекаторы%проесса)
- Переменные окружения
- Текущий каталог (PWD).

Если "подмена" нужна единажды. Для примен:`ls 2>&1` тут команда exec не используется и перенаправление потока происходит пока команда `ls` не завершится.

Команда `exec` выполняет три основные функции:

1. Постоянное перенаправление (Модификация текущей оболочки).
	   Модификация может быть следующая:
		 - **Открыть новый дескриптор на запись**: `exec 5> file.txt`. Теперь все что пойдёт на 5 дискриптор, запишется в этот файл.
		- **Открыть на чтение**. `exec 5< file.txt`. Теперь, обращаясь к 5 дискриптеру можно читать данные с файла.
		- **Перенаправить стандартный вывод** `exec 1> log.txt` После этого весь вывод в терминале будет записываться в файл.
2. Закрытие дескриптера.
	1. Если канал больше не нужен, его можно закрыть, чтобы не тратить ресурсы системы, например. Или не занимать таблицу дескриптеров
		1. **Команда:** `exec 5>&`. Фактически, номер пять удалится из таблицы дескриптеров.
3. Полная замена процесса **ОПИСАНА ВНАЧАЛЕ ЗАМЕТКИ**
	1. Пример `exec htop`

## Шпаргалка по символам (чтобы не путаться):

| Символ | Что делает                                                |
| ------ | --------------------------------------------------------- |
| `>`    | Направить в **файл** (создать/перезаписать)               |
| `>>`   | Дописать в конец **файла**                                |
| `<`    | Взять данные из **файла**                                 |
| `>&`   | Направить один **дескриптор** в другой (например, `1>&2`) |
| `<&`   | Взять ввод из другого **дескриптора**                     |
| `&-`   | **Закрыть** дескриптор                                    |
|        |                                                           |
