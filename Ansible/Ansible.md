**Ansible** - это система для автоматизации ручной работы по настройки или обновлению оборудования (сервера, сеть, пользовательские ПК, VM).

**Ansible** - работает через SSH подключение.

Принцип работы:
	Существует Ansible сервер. На сервере лежат скрипты (они же playbooks) которые описывают, что нужно делать на удаленной машине. Плейбуки необходимы, чтобы их запустить. Когда мы запускаем плейбук - он подключается к удалённой машине\машинам и выполняет определенные действия по настройки\обновлению.

Как ansible понимает с какой машиной работать? Данные об оборудовании хранятся в **inventory-файлах**

Управляемые **ansible'ом** машины могут быть на:
- Linux: Нужен - (Username/Password или SSH Key) + Python 2.6+
- Windows: Нужен - Username/Password + PowerShell 3.0+

Используемые порты:

- Linux - ssh port (по умолчанию 22)
- Windows - WinRM port (по умолчанию 5986) (для ansible нужен python пакет для работы с winrm (pip install pywinrm) (pipx inject ansible pywinrm), и на серверах windows по умолчанию winrm отключен, включается и настраивается скриптом ниже)
