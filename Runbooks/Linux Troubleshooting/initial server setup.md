### Connection

Изначально, когда голая виртуалка поднята, на неё происходит заход под root, через ssh по паролю.

Исходя из принципов безопасности, не стоит оставлять заход на сервак под root - то самый очевидный и главный вектор атаки на сервера, плюс это небезопасно в работе. Если работать под рут, всегда можно что-то сломать.

Подключаемся по ssh с паролем. На локальном генерируем ssh-ключ `ssh-keygen -t ed25519 -c "your@mail"`.

Далее, для более безопасного подключения по ssh необходимо использовать подключение к серваку через ключ. Для этого необходимо будет передать [публичный ключ](SSH) на сервер.

### Users initialization
Далее, первый шаг в поднятии сервера - необходимо создать роли на нём. Вспоминаем принцип [Least Privilege](Least%20Privilege).

Создание разных ролей, с разными павами, доступами - необходимо для разграничения работы, а так же, чтобы можно было отследить, что и где сделал. 

После создания первой роли - админской, нашей роли, необходимо дать sudo доступ нашей роли.

После того, как мы дали sudo доступ нашему юзеру и убедились, что это работает - необходимо закрыть подключение под [root](sudo).
